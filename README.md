# ScriptChecker — как загрузить на Vercel

## Файлы в этом наборе
- `index.html` — сама страница
- `style.css` — оформление
- `script.js` — логика (что происходит при нажатии кнопки)
- `api/analyze.js` — серверная функция, которая проверяет скрипт

## Как загрузить (раз ты уже подключил GitHub к Vercel)

1. Зайди в свой репозиторий на github.com (тот, который подключён к Vercel).
2. Нажми **Add file → Upload files**.
3. Перетащи туда `index.html`, `style.css`, `script.js`.
4. Затем создай папку `api` (при загрузке файла просто напиши в поле имени `api/analyze.js` — GitHub сам создаст папку) и загрузи в неё `analyze.js`.
5. Внизу нажми **Commit changes**.
6. Открой Vercel — он сам увидит изменения в GitHub и через 30–60 секунд выложит новую версию сайта.
7. Открой ссылку своего сайта (что-то вроде `https://твой-проект.vercel.app`) и проверь.

## Важно

- Никакой платы не требуется — весь этот набор работает на бесплатных тарифах Vercel.
- `api/analyze.js` работает как отдельная маленькая программа на сервере — Vercel запускает её автоматически, ничего дополнительно настраивать не нужно.
- Функция сама скачивает код по ссылке (если это `loadstring(game:HttpGet(...))`) и ищет подозрительные признаки — код при этом пользователю обратно не показывается, только описание того, что скрипт делает.

## Если захочешь добавить новые проверки

Открой `api/analyze.js` и добавь новый объект в массив `RULES` — по образцу уже существующих. Не забудь описание писать понятным языком, а не техническим.
